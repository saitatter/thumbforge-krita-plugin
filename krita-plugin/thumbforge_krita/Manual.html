Thumbforge

Thumbforge batch-exports thumbnail variants from a saved Krita .kra template.

Basic workflow

  1. Open and save a .kra template in Krita.
  2. Open Settings > Dockers > Thumbforge.
  3. Click Detect Text to scan vector text shapes.
  4. Map each detected text shape to a variable name.
  5. Import CSV data, paste rows, or add rows manually.
  6. Set a filename pattern such as thumb_{episode}.
  7. Click Save Setup to store the configuration inside the .kra file.
  8. Use Preview Row, Export Current, Export Selected, or Export All.

Filename patterns

  • thumb_{episode}
  • {series}/thumb_{episode}
  • {title}

Thumbforge sanitizes invalid filename characters and adds numeric suffixes for duplicates.

Export settings

  • Formats: PNG, JPG, WebP
  • Presets: YouTube PNG, Small PNG, Transparent PNG
  • Optional resize, compression, quality, alpha, sRGB, and ICC settings

Notes

  • Thumbforge stores setup data inside the .kra document annotation.
  • Save the template before export so Krita uses the latest document state.
  • Use Refresh Text if vector text shapes changed after the initial scan.

Troubleshooting

  • If the docker is missing, restart Krita after enabling the plugin.
  • If text does not update, verify the mapping row and variable values.
  • If export fails, check %APPDATA%\krita\thumbforge.log.
